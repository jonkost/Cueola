# Cueola App Check Rollout

This repo now has an optional Firebase App Check bootstrap in `index.html` and
`dashboard.html`. It is intentionally disabled until a real reCAPTCHA v3 site key
is added.

## Current Code State

- `APP_CHECK_RECAPTCHA_V3_SITE_KEY` is empty in both app entry points.
- When the key is empty, Firebase still starts as it does today.
- When the key is set, the pages initialize App Check before Firestore is used.
- `APP_CHECK_DEBUG_LOCAL` is `false` so no debug token can be generated by
  accident.

## Rollout Order

1. In Firebase Console, go to Security > App Check.
2. Register the Cueola web app for App Check with reCAPTCHA v3.
3. Add the production domains that should be allowed by the reCAPTCHA key.
4. Copy the public reCAPTCHA v3 site key into both:
   - `index.html`
   - `dashboard.html`
5. Test the hosted app with Firestore App Check enforcement still off.
6. Watch App Check request metrics until normal app traffic is verified.
7. For local development only, set `APP_CHECK_DEBUG_LOCAL` to `true`, run from
   `localhost`, copy the browser console debug token, and register that token in
   Firebase Console. Do not commit a debug token.
8. Enable enforcement for Cloud Firestore only after the hosted app and dashboard
   both show verified requests.

## Important Warnings

- Do not enable Firestore enforcement while the site key is blank.
- Do not use `file://` as the validation surface for App Check. Use hosted
  domains for production and `localhost` with a registered debug token for local
  work.
- App Check reduces abuse from non-app clients, but it does not identify users.
  Session codes still grant broad access to each shared session until Firebase
  Auth and role-backed rules are added.
- Firestore rules should remain compatible with shared-code sessions until an
  auth migration is designed and tested.

## After App Check

The next security layer is Firebase Auth, likely anonymous auth first, then a
session role model that can distinguish:

- Viewer
- Rundown editor
- Planda Bear editor
- Flowmingo operator
- Instructor/admin
