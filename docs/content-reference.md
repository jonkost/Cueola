# Cueola Content Reference

These reference IDs power Cueola Learning Hub voice-over. The browser plays `assets/narration/af_heart/{referenceId}.mp3` when the reference appears in the manifest. The Cueola voice-over path is Kokoro-only, so missing files show as pending instead of using browser speech.

| Reference ID | Area | Narration Text |
| --- | --- | --- |
| `UI-vo-preview.prompt` | Voice Over | Voice over is on. Open the guide to hear each lesson. |
| `LH-start.lesson` | Learning Hub | What The Three Tools Do. Cueola is the show brain. Planda Bear is the prep package. Flowmingo is the script display and remote prompter. Step 1. Use Cueola to create or join a show, build rows, add cues, and run the live view. Step 2. Use Planda Bear before show day for call sheets, schedules, safety plans, patch sheets, comments, and PDF exports. Step 3. Use Flowmingo when talent needs a clean script screen that can be controlled from another window or device. Step 4. Keep one shared session code. That code is the bridge between the rundown, paperwork, and prompter. Session code. The same code connects collaborators, Planda Bear, Flowmingo Talent Display, and Flowmingo Remote Op. Best first move. Load Demo, open this guide beside it, and walk through the lessons once before using a real show. |
| `LH-cueola-build.lesson` | Learning Hub | Build A Rundown. A Cueola rundown is built from rows. Each row is one beat in the production, and each cue cell holds the instructions for one department. Step 1. Join a session or create a Blank Slate code. Step 2. Add a row for each show beat: open, intro, segment, package, conversation, outro, or any custom label. Step 3. Choose Timed when the row has a known duration. Choose Flex when it can breathe. Step 4. Click any cue cell to add Video, Audio, Playback, GFX, Lighting, or Script instructions. Step 5. Use Edit when you need to reorder rows, clean labels, or remove old beats. Rows are beats. Think in production moments, not spreadsheet lines. One row should tell the room what happens next. Script feeds Flowmingo. Anything typed into a Script cue can be assembled and pushed to the talent display. |
| `LH-cueola-live.lesson` | Learning Hub | Run Live Without Panic. Live mode turns the rundown into a show-caller surface. It highlights now, next, remaining time, and Flowmingo connection state. Step 1. Press Go Live after your rows and script cues are ready. Step 2. Review the pre-live checks for script, Flowmingo talent, and cloud sync. Step 3. Use Prev and Next to move the show one row at a time. Step 4. Use the Script Op panel to edit the live script and push updates to Flowmingo. Step 5. Watch the Flowmingo status badge. Connected means talent is sending heartbeats; applied controls confirm the remote worked. Operator view. Flowmingo Op mode focuses the live screen on prompter controls and script preview. Remote confidence. A sent command is not enough. The app now waits for the talent display to acknowledge applied controls. |
| `LH-plandabear.lesson` | Learning Hub | Prep The Show Package. Planda Bear keeps the production paperwork in the same workspace as the rundown so the team is not chasing separate files. Step 1. Open Planda Bear from the home screen or the topbar. Step 2. Start with the Call Sheet: production name, date, call time, location, contacts, access, crew, and talent. Step 3. Fill the Production Schedule and readiness checklist so setup, rehearsal, show, and wrap are clear. Step 4. Add Safety Plan details before the room gets busy. Step 5. Use Video Patch, Audio Patch, and Comms Patch sheets to document routing. Step 6. Preview or export the PDF package when the paperwork is ready to share. Comments. Instructors can leave Planda Bear comments without overwriting student work. One package. Export One PDF Package gathers the paperwork and rendered rundown into a shareable file. |
| `LH-flowmingo-talent.lesson` | Learning Hub | Set Up The Talent Display. The Talent Display is the screen talent reads. It should be opened in its own browser window or device and controlled remotely. Step 1. Open Flowmingo Talent Display from the home screen or live screen. Step 2. Enter the same session code used by the show. Step 3. Wait for the READY state and confirm the Script check is on. Step 4. Use Controls only for local setup: speed, size, alignment, theme, mirror, and fullscreen. Step 5. Once live, leave the talent display alone. Run it from Script Op or Flowmingo Remote Op. Do not touch the talent window. The remote path is built so the operator can control play, pause, speed, size, theme, mirror, and reset without focusing the talent screen. Break markers. Markers like BREAK AUTO PAUSE and STOP HERE can help the prompter stop at planned moments. |
| `LH-flowmingo-remote.lesson` | Learning Hub | Run The Remote Prompter. Flowmingo Remote Op is the dedicated control surface for the talent display. It is meant to work from another tab, window, or device. Step 1. Open Remote Op and load the same session code as the talent screen. Step 2. Confirm the status says ready, then use Play to start talent scrolling. Step 3. Use Space for play and pause, hold Down to brake, hold Up to boost, Left and Right for text size, and Option plus Down or Up for direction. Step 4. Use Reset if the talent needs to return to the top. Step 5. Watch for applied acknowledgements. If a command does not ack, check that the talent display is still connected. Control ownership. If Flowmingo Remote Op is active, Script Op pauses its own remote control briefly so operators do not fight each other. Trouble signal. No talent ack means the command was sent but the talent screen did not confirm it. Reconnect the talent display or reload its session code. |
| `LH-support.lesson` | Learning Hub | First Fixes When Something Feels Off. Most show-day problems come from code mismatch, a missing script, or a talent screen that was closed, reloaded, or asleep. Step 1. If collaborators cannot see the show, confirm everyone has the same session code. Step 2. If Flowmingo is blank, confirm there is Script cue text and push to Flowmingo from live Script Op. Step 3. If the remote says sent but not applied, reload the Talent Display and enter the same code again. Step 4. If the wrong row is live, use Prev or Next until Now matches the room. Step 5. If paperwork looks stale, reopen Planda Bear and check the activity and comments area. Fast rehearsal check. Before doors open: open Talent Display, open Remote Op, press Play, Reset, Mirror, then Reset again. When in doubt. A clean reload plus the same session code should reconnect the show, paperwork, and prompter surfaces. |
